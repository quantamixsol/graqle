🔧 Built something for the Python community — would love feedback!

GraQle is a Python SDK that builds knowledge graphs from codebases. Instead of AI tools reading 60 files to answer "what depends on auth?", they query the graph — getting answers in 500 tokens instead of 50,000.

Quick start:
pip install graqle
graq scan repo .
graq run "what depends on auth?"

Features:
• Dependency analysis + impact analysis
• MCP server (16 tools) for Claude Code, Cursor, VS Code
• 14 LLM backends (including Ollama for fully offline use)
• Institutional memory — the graph remembers what your team learns
• 2,000+ tests, v0.31.3 on PyPI

GitHub: https://github.com/quantamixsol/graqle
PyPI: https://pypi.org/project/graqle/

What would you want your AI coding assistant to understand about your codebase? Curious what features resonate most!

Built a tool that makes AI coding assistants architecturally aware.

Problem: AI tools read files, not architecture. They burn 50K tokens to answer a dependency question that a graph traversal answers in 500 tokens.

Solution: GraQle builds a knowledge graph from your codebase and exposes it as an MCP server. Your AI queries the graph for precise, structured context.

The approach:
- NetworkX-based knowledge graph
- Multi-agent graph reasoning
- Task routing across 14 LLM backends
- Confidence scoring with evidence chains
- Two EU patents filed on the reasoning approach

Try it: pip install graqle

GitHub: https://github.com/quantamixsol/graqle

Interested in feedback from the AI community on the graph-based reasoning approach!
